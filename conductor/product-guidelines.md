# Product Guidelines - Open Data Lakehouse Demo

## Prose Style
- **Educational & Friendly:** Documentation should be encouraging and easy to follow, making complex concepts accessible to a broad audience.
- **Deep Technical:** While being friendly, we don't sacrifice depth. All technical explanations must be precise, detailed, and accurate.
- **Tone:** Professional, informative, and proactive. Use active voice and clear headings.

## Branding & Visual Identity
- **GCP Centric:** The project should clearly highlight the strengths of the Google Cloud ecosystem, integrating seamlessly with its services (BigQuery, Dataproc, GCS).
- **Modern & Clean:** Visual elements (in the dashboard) should follow a clean, modern tech aesthetic.
- **Consistency:** Use consistent terminology for GCP services and open-source components.

## UX Principles
- **Simplicity First:** The barrier to entry for running the demo should be as low as possible.
- **Guided Experience:** Users should be led through the data journey with clear instructions, helpful tooltips, and logical progression.
- **Feedback Loops:** Provide immediate feedback for user actions (e.g., system status indicators in the dashboard).
